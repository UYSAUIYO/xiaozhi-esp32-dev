# 十四、调试日志增强规则

当 Bug 缺少日志时，应优先添加分层日志。

BSP 层日志应关注：

1. 初始化是否成功；
2. 总线是否成功；
3. 设备地址是否响应；
4. 寄存器读写是否成功；
5. GPIO 状态是否正确；
6. 硬件错误码。

Middleware 层日志应关注：

1. 状态切换；
2. 配置加载；
3. 任务创建；
4. 事件触发；
5. 数据处理结果；
6. 异常状态。

App 层日志应关注：

1. 服务启动顺序；
2. 功能启停；
3. 用户请求；
4. 业务流程状态；
5. 关键错误汇总。

日志 TAG 应与模块名一致，不应所有文件共用一个无意义 TAG。
