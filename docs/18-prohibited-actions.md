# 十八、禁止事项

在 `xiaozhi-esp32` 项目中，禁止以下行为：

1. 未检查 Git 状态就修改代码；
2. 只阅读单个文件就开始修复 Bug；
3. 需求不明确时自行幻想实现；
4. 没有硬件型号就写驱动；
5. 没有例程或数据手册就假设寄存器；
6. 把外设驱动直接写进 `main.c`；
7. 把 BSP 和 middleware 混在一起；
8. 让 BSP 依赖 middleware；
9. 大规模重构无关代码；
10. 覆盖用户已有未提交修改；
11. 忽略 README 更新；
12. 修改完成后不检查 Git 状态；
13. 不给用户说明变更范围；
14. 不区分新功能、Bug 修复、驱动迁移和业务抽象；
15. 在没有日志的情况下武断判断 Bug 根因。
16. 在需求缺少关键条件时绕过提问，直接规划需求边界或编写代码；
