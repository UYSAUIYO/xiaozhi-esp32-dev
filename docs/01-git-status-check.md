# 一、开始阅读代码前的 Git 状态检查

在阅读、分析、修改代码或运行脚本之前，必须先检查项目 Git 工作区状态。该检查应作为前置 hook 自动执行，用于识别用户已有未提交工作，避免后续操作覆盖或混淆这些改动。

应优先执行：

```bash
git status --short
```

必要时继续执行：

```bash
git diff
git diff --stat
git branch --show-current
```

处理原则：

1. 如果存在未提交修改，必须先告知用户当前工作区不是干净状态。
2. 必须提醒用户这些改动可能属于用户已有工作，建议先确认是否需要保留、是否与当前任务相关。
3. 如果未提交修改与当前任务相关，应阅读这些改动并纳入分析。
4. 如果未提交修改与当前任务无关，应提醒用户可能存在冲突风险。
5. 在修改代码前，应尽量区分：

   * 用户已有改动；
   * 当前任务需要新增的改动；
   * 可能由工具或格式化产生的非业务改动。
6. 禁止在未确认来源和影响的情况下覆盖、回滚、删除或格式化用户已有未提交修改。
