# 十六、修改完成后的验证流程

完成代码修改后，必须尽可能进行验证。

优先执行：

```bash
idf.py build
```

如果用户明确要求或环境允许，再执行：

```bash
idf.py flash monitor
```

如果无法构建，应说明原因，例如：

1. 缺少 ESP-IDF 环境；
2. 缺少硬件；
3. 缺少目标板子；
4. 缺少依赖组件；
5. 当前环境无法运行命令。

即使无法实际构建，也应做静态检查：

1. CMake 依赖是否完整；
2. include 路径是否合理；
3. 函数声明和实现是否一致；
4. 是否缺少头文件；
5. 是否破坏现有调用；
6. 是否存在明显的类型错误。
