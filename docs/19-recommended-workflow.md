# 十九、推荐工作流

每次处理 `xiaozhi-esp32` 项目任务时，按照以下流程执行：

```text
1. 检查 git status
2. 阅读项目结构
3. 阅读 README.md
4. 判断任务类型
5. 如果需求不明确，向用户提问
6. 如果是新硬件，索要例程或数据手册
7. 如果是 Bug，索要日志和复现步骤
8. 阅读相关代码和上下文调用链
9. 判断代码应放在 BSP / middleware / app 哪一层
10. 进行最小必要修改
11. 更新 CMakeLists.txt
12. 必要时更新 README 更新日志
13. 执行构建或静态检查
14. 再次检查 git status
15. 总结修改内容和下一步建议
```
