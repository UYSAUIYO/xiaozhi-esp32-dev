# 十三、新硬件驱动迁移流程

当用户提出新硬件驱动编写或加入新硬件支持时，应先触发新硬件资料确认 hook：

```text
先确认是否存在官方例程、Arduino 例程、ESP-IDF 例程、寄存器手册、通信协议说明，再开始迁移代码。
```

如果用户已经提供硬件例程，应按以下步骤迁移：

1. 阅读例程目录结构；
2. 找到初始化入口；
3. 找到通信接口；
4. 找到寄存器读写函数；
5. 找到核心配置流程；
6. 找到数据读取或控制函数；
7. 判断是否依赖裸机、Arduino、STM32 HAL 或其他 SDK；
8. 将底层通信适配到 ESP-IDF；
9. 将基础驱动放入 BSP；
10. 将业务抽象放入 middleware；
11. 编写清晰的头文件 API；
12. 添加 ESP_LOG 日志；
13. 添加错误处理；
14. 修改 CMake 依赖；
15. 在目标板子主代码中最小化接入；
16. 构建验证。

迁移时不应直接复制无关平台代码，应保留逻辑，替换平台相关接口。

如果用户没有例程代码，必须先索要数据手册、寄存器手册或通信协议说明；如果例程和资料都没有，禁止直接编写驱动实现。
