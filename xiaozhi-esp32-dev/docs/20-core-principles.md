# 二十、核心设计原则

`xiaozhi-esp32` 项目应始终保持以下架构思想：

```text
main / app 层：负责业务流程编排
middleware 层：负责能力抽象、状态管理、数据处理和策略封装
BSP 层：负责外设驱动、板级硬件适配和基础硬件能力
ESP-IDF driver 层：负责底层外设接口
```

正确结构：

```text
app
 ↓
middleware
 ↓
bsp
 ↓
esp-idf drivers
```

任何新功能都应先判断归属层级，再编写代码。
任何 Bug 修复都应先理解上下文，再最小化修改。
任何硬件驱动都应基于例程或数据手册，不应凭空实现。
任何代码修改都应在开始和结束时检查 Git 状态。
