# 十三、新硬件驱动迁移流程

当用户提供硬件例程后，应按以下步骤迁移：

1. 阅读例程目录结构；
2. 找到初始化入口；
3. 找到通信接口；
4. 找到寄存器读写函数；
5. 找到核心配置流程；
6. 找到数据读取或控制函数；
7. 判断是否依赖裸机、Arduino、STM32 HAL 或其他 SDK；
8. 将底层通信适配到 ESP-IDF；
9. 将基础驱动放入 BSP；
10. 将业务抽象放入 middleware；
11. 编写清晰的头文件 API；
12. 添加 ESP_LOG 日志；
13. 添加错误处理；
14. 修改 CMake 依赖；
15. 在目标板子主代码中最小化接入；
16. 构建验证。

迁移时不应直接复制无关平台代码，应保留逻辑，替换平台相关接口。
