# 一、开始阅读代码前的 Git 状态检查

在阅读、分析或修改代码之前，必须先检查项目 Git 工作区状态。

应优先执行：

```bash
git status --short
```

必要时继续执行：

```bash
git diff
git diff --stat
git branch --show-current
```

处理原则：

1. 如果存在未提交修改，必须先告知用户当前工作区不是干净状态。
2. 不应直接覆盖用户已有改动。
3. 如果未提交修改与当前任务相关，应阅读这些改动并纳入分析。
4. 如果未提交修改与当前任务无关，应提醒用户可能存在冲突风险。
5. 在修改代码前，应尽量区分：

   * 用户已有改动；
   * 当前任务需要新增的改动；
   * 可能由工具或格式化产生的非业务改动。
